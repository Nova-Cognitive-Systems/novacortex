# RFC PMF-001: Portable Memory Format

**Status:** Draft
**Version:** 1.0
**Author:** NovaCortex Team
**Created:** 2026-03-30
**URI:** https://novacortex.dev/rfc/pmf-001

## Abstract

This document specifies the Portable Memory Format (PMF), a JSON-based interchange format for cognitive memory systems. PMF enables complete portability of memory graphs across NovaCortex instances, including semantic relationships, embedding vectors, and integrity verification.

## Motivation

Existing export formats (plain JSON) lose critical information:
- Graph topology and relation weights
- Embedding vectors for semantic search continuity
- Federation metadata for distributed deployments
- Integrity verification for data validation

PMF addresses these gaps while remaining human-readable and tooling-friendly.

## Format Specification

### File Extension

`.pmf.json`

### MIME Type

`application/vnd.novacortex.pmf+json`

### Magic Identifier

The `header.magic` field MUST be `"NCPMF"` (NovaCortex Portable Memory Format).

### Structure

```json
{
  "header": { ... },
  "graph": { ... },
  "memories": [ ... ],
  "relations": [ ... ]
}
```

## Header Object

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `magic` | string | Yes | Must be `"NCPMF"` |
| `version` | string | Yes | Format version, currently `"1.0"` |
| `created` | ISO 8601 | Yes | Export timestamp |
| `source` | object | Yes | Source metadata |
| `integrity` | object | Yes | Verification data |

### Source Object

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `namespace` | string | Yes | Source namespace |
| `nodeId` | string | No | Federated node identifier |
| `exportedBy` | string | No | Agent or user ID |
| `description` | string | No | Human-readable description |

### Integrity Object

| Field | Type | Description |
|-------|------|-------------|
| `memoryCount` | integer | Number of memories in payload |
| `relationCount` | integer | Number of relations in payload |
| `embeddingDim` | integer | Embedding vector dimension (0 if not included) |
| `merkleRoot` | string | SHA-256 of sorted contentHash values |
| `checksum` | string | CRC32 of `{memories, relations}` payload (hex) |

## Graph Metadata Object

Provides topology information without requiring full graph traversal:

| Field | Type | Description |
|-------|------|-------------|
| `nodes` | integer | Number of memory nodes |
| `edges` | integer | Number of relations |
| `density` | float | `edges / (nodes * (nodes - 1))` |
| `components` | integer | Number of disconnected subgraphs |
| `avgDegree` | float | Average relations per memory |
| `hubNodes` | string[] | Top 5 most-connected memory IDs |

## Memory Entry Object

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | Yes | Memory UUID |
| `namespace` | string | Yes | Namespace identifier |
| `content` | string | Yes | Memory content |
| `contentHash` | string | Yes | SHA-256 of content |
| `memoryType` | enum | Yes | `episodic`, `semantic`, `procedural`, `working` |
| `created` | ISO 8601 | Yes | Creation timestamp |
| `accessed` | ISO 8601 | Yes | Last access timestamp |
| `version` | integer | Yes | Version counter |
| `metadata` | object | Yes | Memory metadata |
| `embedding` | float[] | No | Vector embedding |

### Metadata Object

| Field | Type | Description |
|-------|------|-------------|
| `confidence` | float | 0.0 - 1.0 |
| `salience` | float | 0.0 - 10.0 |
| `decayRate` | float | Decay coefficient |
| `effectiveSalience` | float | Computed salience after decay |
| `tags` | string[] | Classification tags |
| `entities` | Entity[] | Extracted entities |
| `signals` | Signal[] | Keyword signals |
| `source` | Source | Origin metadata |

## Relation Entry Object

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | Yes | Relation UUID |
| `from` | string | Yes | Source memory ID |
| `fromNs` | string | Yes | Source namespace |
| `to` | string | Yes | Target memory ID |
| `toNs` | string | Yes | Target namespace |
| `type` | enum | Yes | Relation type |
| `strength` | float | Yes | 0.0 - 1.0 |
| `bidirectional` | boolean | Yes | Two-way relation |
| `metadata` | object | No | Custom metadata |
| `created` | ISO 8601 | Yes | Creation timestamp |

### Relation Types

- `causes` / `caused_by`
- `related_to`
- `contradicts`
- `supports`
- `supersedes`
- `part_of`
- `references`
- `temporal_before` / `temporal_after`

## Integrity Verification

### Merkle Root Calculation

```
1. Extract contentHash from each memory
2. Sort hashes lexicographically
3. Concatenate sorted hashes
4. SHA-256 hash the concatenation
```

### Checksum Calculation

```
1. Serialize {memories, relations} as JSON (no whitespace)
2. Compute CRC32
3. Output as 8-character lowercase hex
```

## Import Behavior

When importing a PMF file:

1. Verify `header.magic === "NCPMF"`
2. Verify Merkle root matches computed value
3. Verify CRC32 checksum
4. Skip existing memories (by ID + namespace)
5. Create new memories with preserved metadata
6. Recreate relations

## Security Considerations

- PMF files may contain sensitive data; encrypt at rest
- Validate checksums before importing untrusted files
- Sanitize content fields to prevent injection attacks
- Embeddings can potentially be reversed; consider omitting for public exports

## Future Extensions

- Binary PMF format (`.pmf`) for reduced file size
- Differential exports (only changed memories)
- Encrypted payload option
- Streaming format for large exports
